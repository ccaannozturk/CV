# Can Öztürk - Personal Portfolio & Interactive CV

**[🌐 View Live Portfolio](https://ccaannozturk.github.io/CV/)**

This repository contains the source code for my personal portfolio website. It is designed to showcase my journey as a **Product Financial Controller**, bridging the gap between traditional finance, modern technology, and AI-driven automation.

## 🎯 Purpose

The goal of this project was to create a modern, high-performance, and responsive single-page application that reflects my professional identity: a finance expert who speaks the language of product teams and leverages AI/Automation to drive efficiency.

## 🚀 Key Features

* **Responsive Architecture:** Fully optimized for all screen sizes using Tailwind CSS.
* **Interactive Career Timeline:** Dynamic modals to showcase detailed responsibilities and achievements at Adyen, Vivo Energy, KPMG, and EY.
* **AI-Enhanced Professionalism:** Highlights my unique approach to "vibecoding," AI agent orchestration, and automated financial insights.
* **Modern UI/UX:** Clean navigation, scroll-triggered animations (`IntersectionObserver`), and a professional color palette.

## 🛠️ Tech Stack

* **Frontend:** HTML5, Tailwind CSS (Utility-first styling).
* **Interactivity:** Vanilla JavaScript (Modal logic, mobile menu, and scroll effects).
* **Design:** Google Fonts (Montserrat & Lato).
* **Hosting:** GitHub Pages.

## 🤖 The "Finance + Tech" Edge

Unlike traditional financial portfolios, this site highlights a tech-forward mindset:
* **AI Collaboration:** Mastery in using AI agents to deconstruct complex workloads.
* **Vibecoding:** Rapid prototyping and adaptive development.
* **Process Automation:** Transforming manual financial tasks into streamlined digital pipelines.

## 🏃‍♂️ How to Run Locally

While the project is live on GitHub Pages, you can also run it locally:

1.  Clone the repository:
    ```bash
    git clone [https://github.com/ccaannozturk/CV.git](https://github.com/ccaannozturk/CV.git)
    ```
2.  Open `index.html` in your preferred browser.
3.  No dependencies or build steps required—just pure, efficient code.
